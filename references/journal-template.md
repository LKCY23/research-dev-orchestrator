# JOURNAL.md Template

`JOURNAL.md` is the append-only human-readable session memory for a run. It is required for long-running, cross-week work.

Append one concise entry at the end of every working session.

```markdown
## 2026-07-03T12:00:00Z Session

### Summary

One concise sentence describing this session.

### What Changed

- 

### Decisions / Tradeoffs

- 

### Experiment Notes

- 

### Current State

- Valid protocol state:
- Task counts:
- Ready for review:
- Blocked:

### Next Actions

- 
```

Use `JOURNAL.md` for session context, non-architecture tradeoffs, user preferences, and next actions. Use ADRs only for durable architecture or design decisions. Use `RESULT_LEDGER.md` for experiment results and claim support.
